2.​ Preguntas Referidas al Sistema VNRGIS.​
​ ​PREGUNTA ​RESPUESTA
​ ​Cuando se intenta instalar la base de datos del sistema aparece el siguiente mensaje: No se puede utilizar el archivo backup <archivo> porque fue originalmente formateado con tamaño de sector 512 y ahora se tiene un dispositivo con sector de tamaño 2048.​ ​Este error se produce si se utiliza SQL Server Versión 7. No ocurre utilizando SQL Server 2000. En caso de que no pueda cambiarse de versión, intentar hacer la instalación copiando los archivos instaladores a un disco y no hacerla directamente del CD de instalación.​
​ ​Según el Manual de Usuario, el sistema requiere que el usuario "sa" no tenga password. ¿Es posible definir un usuario con password para el Sistema?​​ ​Si. Las empresas que requieran ésta funcionalidad deberán solicitar a la GART los archivos de actualización correspondientes.​
​​ ​¿Se puede utilizar SQL Server Version 7, como motor de base de datos del sistema VNRGIS?​​ El sistema VNRGIS ha sido desarrollado para funcionar con el motor de base de datos SQL Server 2000 , por lo que se recomienda que en caso no se cuente con dicha versión del motor de base de datos, se utilice el motor MSDE, el cual viene en el CD de instalación entregado.
